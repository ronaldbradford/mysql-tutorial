# uuidv() for MySQL 8.4

`uuidv(version)` returns a UUID string of the requested version.

Each implementation exposes a distinctly named function so all three can be
loaded simultaneously:

| Function | Implementation | Load mechanism |
|----------|---------------|----------------|
| `uuidv_udf(version)` | UDF plugin | `CREATE FUNCTION` |
| `uuidv_plugin(version)` | daemon server plugin | `INSTALL PLUGIN` + `CREATE FUNCTION` |
| `uuidv_component(version)` | component | `INSTALL COMPONENT` |

Supported versions:

| Argument | Result |
|----------|--------|
| `1` | time-based (Gregorian ticks + random node) |
| `4` | random |
| `6` | reordered time-based (sortable) |
| `7` | Unix-ms time-ordered |

Versions 3 and 5 (name-based) are intentionally rejected — they require a
namespace and name, not just a version number. Any other unsupported version
raises an error.

`uuid_gen.h` contains the shared, self-contained UUID generation logic used by
all three implementations.

## Building

### 1. Get MySQL 8.4 server headers

The server plugin and component headers (`plugin.h`, component service headers)
are not shipped with the MySQL binary packages. Do a sparse clone of just the
`include/` tree — much faster than a full clone:

```bash
git clone --depth=1 --filter=blob:none --sparse \
    --branch 8.4 \
    https://github.com/mysql/mysql-server.git ~/git/mysql-8.4-src

cd ~/git/mysql-8.4-src
git sparse-checkout set include
git checkout

# plugin.h needs sql/sql_plugin.h for C++ builds; fetch it individually
git sparse-checkout add --skip-checks sql/sql_plugin.h
git checkout
```

`plugin.h` also needs `mysql_version.h`, which is generated by CMake in a full
source build. Copy it from your installed MySQL binary instead:

```bash
# Adjust the path if your MySQL is not in Homebrew
cp /opt/homebrew/opt/mysql@8.4/include/mysql/mysql_version.h \
   ~/git/mysql-8.4-src/include/mysql/mysql_version.h
```

### 2. Build all three

A standalone `CMakeLists.txt` at the root of this directory compiles all three
without needing the MySQL build system:

```bash
cd uuidv          # this directory
mkdir -p build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make
```

By default `MYSQL_SRC` points to `~/git/mysql-8.4-src`. Override if needed:

```bash
cmake .. -DMYSQL_SRC=/path/to/mysql-8.4-src
```

### 3. Install

Install the `.so` files into the MySQL `plugin_dir`:

```bash
cmake --install .
# Installs to /opt/homebrew/opt/mysql@8.4/lib/plugin/ by default.
# Override with: cmake .. -DMYSQL_PLUGIN_DIR=/path/to/plugin_dir
```

## Loading and using

### Component (recommended)

The component registers `uuidv_component()` automatically on install:

```sql
INSTALL COMPONENT 'file://component_uuidv';
SELECT uuidv_component(4);
SELECT uuidv_component(7);
UNINSTALL COMPONENT 'file://component_uuidv';
```

### Daemon server plugin

The daemon plugin requires two steps: `INSTALL PLUGIN` makes it visible in
`SHOW PLUGINS`; `CREATE FUNCTION` exposes the UDF from the same `.so`:

```sql
INSTALL PLUGIN uuidv_plugin SONAME 'uuidv_plugin.so';
CREATE FUNCTION uuidv_plugin RETURNS STRING SONAME 'uuidv_plugin.so';
SELECT uuidv_plugin(4);
SELECT uuidv_plugin(7);
DROP FUNCTION uuidv_plugin;
UNINSTALL PLUGIN uuidv_plugin;
```

### UDF plugin

```sql
CREATE FUNCTION uuidv_udf RETURNS STRING SONAME 'uuidv_udf.so';
SELECT uuidv_udf(4);
DROP FUNCTION uuidv_udf;
```

### All three simultaneously

All three can be loaded at the same time because each registers a differently
named function:

```sql
INSTALL COMPONENT 'file://component_uuidv';
INSTALL PLUGIN uuidv_plugin SONAME 'uuidv_plugin.so';
CREATE FUNCTION uuidv_plugin RETURNS STRING SONAME 'uuidv_plugin.so';
CREATE FUNCTION uuidv_udf RETURNS STRING SONAME 'uuidv_udf.so';

SELECT uuidv_component(7), uuidv_plugin(7), uuidv_udf(7);

DROP FUNCTION uuidv_udf;
DROP FUNCTION uuidv_plugin;
UNINSTALL PLUGIN uuidv_plugin;
UNINSTALL COMPONENT 'file://component_uuidv';
```

## Testing

`test/test_uuidv.sql` runs 44 functional tests covering all three
implementations: length, format, version bit, NULL input, unsupported version,
uniqueness, `SHOW PLUGINS` visibility, system variable defaults and session
scope, and simultaneous coexistence.

Run against any MySQL 8.4 instance that has `plugin_dir` set to the directory
where the `.so` files were installed:

```bash
mysql -uroot -f < test/test_uuidv.sql
```

The `-f` flag is required: `UNINSTALL PLUGIN` and `UNINSTALL COMPONENT` have no
`IF EXISTS` clause and emit an error when nothing is loaded (e.g. on the first
run). With `-f` those errors are printed but execution continues.

Every test prints `PASS` or `FAIL`. All should print `PASS`.

## Notes

* All versions coerce the argument to `INT_RESULT`, so `uuidv_udf(4)`,
  `uuidv_udf('4')`, and `uuidv_udf(col)` all work.
* `const_item` is `false` so each row in a query gets a distinct UUID. Use
  `SET @v = uuidv_udf(4)` (not `SELECT uuidv_udf(4) FROM t`) to verify
  uniqueness — the optimizer evaluates constant-argument UDFs only once in
  aggregated queries.
* `maybe_null` is `true` because the function returns `NULL` for `NULL` input
  or an unsupported version.
* The component is the forward-looking interface; the daemon plugin and UDF are
  provided to demonstrate all three MySQL extension mechanisms side by side.
